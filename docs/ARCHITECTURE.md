## 系统概览

AI 五子棋是一个纯前端项目，无需构建工具即可运行。项目采用模块化架构，将不同的功能分离到独立的 JavaScript 文件中。

### 技术栈

| 技术 | 用途 |
|------|------|
| HTML5 | 页面结构 |
| CSS3 | 样式和动画 |
| Canvas API | 棋盘渲染 |
| Web Audio API | 音效系统 |
| LocalStorage | 数据持久化 |

### 设计原则

1. **模块化** - 使用 IIFE 模式封装模块
2. **松耦合** - 模块间通过公开 API 通信
3. **高性能** - 优化 AI 搜索算法和渲染性能
4. **可扩展** - 易于添加新的 AI 模块

## 架构模式

### 模块模式 (Module Pattern)

所有模块使用 IIFE（立即调用函数表达式）封装：

```javascript
const MyModule = (() => {
    'use strict';

    // 私有变量
    let privateState = 0;

    // 私有函数
    function privateMethod() {
        return privateState;
    }

    // 公开 API
    return {
        publicMethod: () => privateMethod()
    };
})();
```

### 发布-订阅模式 (Pub/Sub)

游戏事件使用自定义事件系统：

```javascript
// 发布事件
gameEvent.emit('move', { row: 5, col: 5, piece: 1 });

// 订阅事件
gameEvent.on('move', (data) => {
    console.log('Move made:', data);
});
```

### AI 模块职责

| 模块 | 职责 | 核心算法 |
|------|------|---------|
| `ai.js` | 协调控制 | Minimax + Alpha-Beta 剪枝 |
| `ai-pattern.js` | 模式识别 | 威胁评估、紧急检测 |
| `ai-strategy.js` | 攻防策略 | 攻势推演、防守预判 |
| `ai-tactics.js` | 战术调配 | 动态权重、游戏阶段分析 |
| `ai-extreme.js` | 极限模式 | 深度搜索、多 AI 投票 |
| `ai-utils.js` | 共享工具 | 基础检查函数 |

### 评分系统

AI 使用专业级棋型评分系统：

```javascript
const SCORES = {
    FIVE:           10000000,  // 五连（获胜）
    LIVE_FOUR:      1000000,   // 活四
    RUSH_FOUR_LIVE: 500000,    // 冲四活三
    DOUBLE_FOUR:    200000,    // 双冲四
    LIVE_THREE:     15000,     // 活三
    RUSH_FOUR:      8000,      // 冲四
    SLEEP_THREE:    1000,      // 眠三
    LIVE_TWO:       350,       // 活二
};
```

### VCT/VCF 战术

**VCT (Visualization of Continuous Threat)**：连续进攻
- 检测连续冲四、活三等必杀棋型
- 递归搜索对手防守后的后续进攻

**VCF (Visualization of Continuous Force)**：连续防守
- 预测对手的连续进攻
- 提前布防

### 渲染器选择

```javascript
function drawBoard() {
    if (isRefined()) {
        // 精致模式渲染
        RefinedRenderer.drawBoardBackground(ctx, ...);
        RefinedRenderer.drawPiece(ctx, ...);
    } else {
        // 标准模式渲染
        drawGrid(ctx, ...);
        drawPiece(ctx, ...);
    }
}
```

## 性能优化

### 1. 搜索优化

- **节点限制**: MAX_NODES = 300,000
- **候选预计算**: 只搜索有邻居的空位
- **Alpha-Beta 剪枝**: 减少搜索空间
- **缓存机制**: 避免重复计算

### 2. 渲染优化

- **离屏渲染**: 棋盘背景缓存
- **分层绘制**: 静态层 + 动态层
- **请求动画帧**: RAF 节流

### 3. 内存优化

- **记忆数据限制**: MAX_RECORDS = 200
- **定期清理**: 自动清理过期数据

## 扩展性

### 添加新 AI 模块

1. 创建新模块文件 `src/ai/ai-newmodule.js`
2. 实现 IIFE 模式
3. 在 `GomokuAI.getBestMove()` 中集成
4. 添加共识决策支持

```javascript
// ai-newmodule.js
const AINewModule = (() => {
    return {
        getSuggestion: (board, piece) => { /* ... */ }
    };
})();
```

### 添加新难度级别

在 `DIFFICULTY_CONFIG` 中添加：

```javascript
const DIFFICULTY_CONFIG = {
    1: { depth: 2, candidates: 15 },
    2: { depth: 3, candidates: 20 },
    3: { depth: 4, candidates: 25 },  // 新难度
    4: { depth: 5, candidates: 30 },  // 专家难度
};
```
